Detailed Description

Main program for multithreaded file server.

Usage Requires one command line arg:
  1. port number to use (on this machine).
Protocol (rules for correct communication)
  • Server initialization
    1. Server starts
    2. Server starts Servers' control thread
    3. Server initializes server socket, then blocks on accept()
  • Client initialization
    1. Server must be running and blocking on accept()
    2. Client starts and calls connect() (perhaps within a constructor)
    3. (Server's accept generates server's client socket)
    4. Server sends welcome message to Client
      • ACK indicates success
      • NACK indicates failure, too many clients
      • DONE indicates client should terminate
    5. Client receives welcome message;
      on success, Server allocates and records client thread
  • Service operation
    1. Client sends request message to Server
      Supported service(s):
      • MSG followed by text
    2. Server performs service, then sends response to Client
      • DONE indicates that Client should terminate
  • Client termination
    1. Client sends END message to Server
    2. Client ends;
      Server performs join() on client thread, then deallocates and forgets it.
  • Server termination
    1. Server's control thread detects EXIT condition (e.g., from stdin), then marks Server for termination (contin = 0)
    2. Server's control thread performs Client initialization
    3. Server (main thread) detects termination and performs Client termination on every recorded Client
    4. Server's control thread performs Client termination; Server (main thread) performs join() on Server's control thread
    5. Server ends
